Transformer

  前面我们讲到 Attention 和 Self-Attention，接下来我们继续学习 Transformer。Transformer 主要包含四个模块：Input、Output、Encoder、Decoder。

Input

  在 Input 模块中包括两个部分，一个是 Word Embedding，其目的是将每个 词 编码成 词向量；另一个是 Positional Embedding ，其目的是编码每个 词 在句子中的位置信息。因为在 RNN 结构中使用 Attention 机制的时候，我们是按照单词的顺序串行计算 Attention 的值，在 Self-Attention 中我们使用的是全局信息并行计算 Attention 的值，没有利用单词的顺序信息，但是单词的位置信息是 NLP 中很重要的一部分，所以在 Transformer 中使用 Positional Encoding 保存单词在序列中的相对或绝对位置的信息。

Word Embedding

  计算机无法跟人类一样轻易的理解单词，我们必须将单词变成计算机能理解的数字，即特征向量 ，而 Word Embedding 就是将每个单词编码成 词向量 的技术统称，其中有很多种方法（model）将词编码成 词向量 ，如 Word2Vec、Glove等。具体操作后面的分享中我们会单独介绍。

  我们了解完 Transformer 中的 Positional Embedding ，我们会有几个疑问：

  1. 我们为什么要这么编码？

  一般情况下，我们都会想到按照 [1,2⋯T]\left[1,2\cdots T\right][1,2⋯T] 的顺序给每个位置编码，但是这样可能会导致后面的位置编码数值过大，且当遇到句子长度超过训练集最长的句子，模型的泛化能力就会变得很差。可能还会想到将所有的位置编码都控制在 [0,1]\left[0,1\right][0,1] 内，这样就可以保证数字不会很大，但是这样会导致不同长度句子的位置编码的步长是不一致的，会影响模型的学习。我们希望位置编码满足以下几点：

  • 在一个句子中，每个词对应的位置编码是唯一的
  • 两个词之间的距离大小(相对位置)和句子的长度无关
  • 可以适用于超过训练时句子的长度,它的值是有界的(不会随着句子加长数字就无限增大 )

  第一个问题我们很自然的想到 词向量 ，我们将每个单词的位置同样编码成一个向量，这样不仅唯一表示每个单词位置，而且方便我们将位置编码和词向量整合在一起。同样，第二个我们会自然的联想到数学中的 周期函数，且是有界的，而数学中最常用到的 有界周期函数 就是正余弦函数。第三个问题就需要我们在周期函数的基础上加以改进，使得满足可以适用于任何长度的句子。

Encoder

  在得到每个单词的 Word Embedding 和 Positional Embedding（两者维度相同）后，论文中将两者想加得到 Transformer 的输入，输入到 Encoder 结构中。下面我们对 Encoder 详细介绍。

  • Multi-Head Attention

  前面我们讲到，Self-Attention 的一个作用是可以实现并行计算，加快模型训练速度，那 Multi-Head Attention 是用来干什么的，有什么用呢？在这里，我们给出一种猜测：我们类比 CNN 中 卷积核 的作用，在图像问题中，我们使用不同的 卷积核 去抓取同一块图像里面的不同信息，可以抓捕到更丰富的图片特征和信息。Transformer 采用 Multi-Head Attention 是希望不同的头关注不同的信息，从而捕捉到更丰富的特征。

  值得注意的是，多头中的每个头 headi⁡\operatorname{head_i}headi​ 的 Q、K、VQ、K、VQ、K、V 有两种情况：

  一种是所有的 headi⁡\operatorname{head_i}headi​ 的 Q、K、VQ、K、VQ、K、V 都是相同的，然后得到多个并列的 Attention 值，再将多个Attention 值拼接起来，再做线性变化使得 Multi-Head Attention 输出维度等于输入维度

  另一种是每个头 headi⁡\operatorname{head_i}headi​ 的 Q、K、VQ、K、VQ、K、V 都是初始 Q、K、VQ、K、VQ、K、V 的一种线性变化，其维度是原始维度的 1h\frac {1}{h}h1​ ,同样也是得到多个并列的 Attention 值，再将多个Attention 值拼接起来，再做线性变化使得 Multi-Head Attention 输出维度等于输入维度。论文中使用的是第二种。

Add & Normal

  Trasformer 的输入经过 Multi-Head Attention后，学习到每个单词之间的相关性和相关程度，并输出每个单词的新的特征表示，这些新的特征表示比原有的特征表示包含更多的特征信息。在进行下一步操作之前，我们需要对新的特征表示进行残差连接和归一化处理，目的是为了防止 梯度消失 和 网络退化，同时可以加快模型的收敛速度。
 

  • Add

  Add 为残差模块,数据在这里进行残差连接。残差连接的思想最经典的代表是 2015 年被提出的 ResNet，主要是用于解决深层网络中出现的网络退化问题。网络退化 是指在增加网络层数的过程中，training accuracy 逐渐趋于饱和，继续增加层数，training accuracy 就会出现下降的现象，而这种下降不是由过拟合造成的。如下图所示：

  为了去解决这个问题，残差连接被提出（其理论推导不是本文的重点，因此不在此介绍），如下图所示：

  • Normal

  Normal 是指 Layer Normalization ,是归一化数据的一种方式，可以有效防止梯度消失或爆炸，同时加快模型收敛速度。(这里不做详细展开讲解，以后会专题讲解)

Feed Forward

  Feed Forward 层比较简单，是一个两层的全连接层，第一层的激活函数为 Relu，第二层不用激活函数，其目的是增强模型的学习能力，对应的公式如下：

FeedForward(X)⁡=Max⁡(0,XW1+b1)W2+b2\operatorname{Feed Forward\left(X\right)} = \operatorname{Max}\left(0,XW_1 + b_1\right)W_2 + b_2 FeedForward(X)=Max(0,XW1​+b1​)W2​+b2​

  以上是 Transformer 的一个 Encoder Block 结构，Transformer 的 Encoder 部分是由多个 Encoder Block 串行连接。Encoder 接收输入矩阵 X∈RN×DX \in \mathbb {R}^{N \times D}X∈RN×D，经过一系列的变换，得到一个输出矩阵 C∈RN×DC \in \mathbb {R}^{N \times D}C∈RN×D，这就是 编码信息矩阵C，类比于 Seq2Seq 的 中间语义向量C。

  

Output

  和前面我们在 Seq2Seq 中讲到解码的时候是按照顺序依次解码,每个时刻的解码输入是由两部分构成,一部分是 中间语义向量C，另一部分是上一时刻的预测值或真实值（使用 Teacher Forcing）。

  可以看到上面的方式是串行的，解码是按照时刻依次解码的，而在 Transformer 中为了实现并行，采用 Mask矩阵 的方式，同时均使用真实标签值去构建 Decoder 的输入，从而实现并行计算。

  • Mask矩阵

  上面我们提到了 Mask矩阵，接下来我们详细讲解一下 Mask矩阵：

  1. 第一步

  输入 Decoder 的输入矩阵以及 Mask矩阵

  2. 第二步

  通过 Decoder 的输入矩阵计算 Self-Attention 的 Q,K,VQ,K,VQ,K,V 矩阵。然后计算 Q⋅KTQ \cdot K^TQ⋅KT

  3. 第三步

  在对 Q⋅KTQ \cdot K^TQ⋅KT 进行 softmax⁡\operatorname{softmax}softmax 之前，我们和 Mask矩阵 做点积，然后在进行 softmax⁡\operatorname{softmax}softmax。每一行和等于 0，但是由于 Mask矩阵，导致单词 0 在单词 1，2，3，4上的 attention 权重都为 0。

  4. 第四步

  使用 Mask⊙Q⋅KTMask \odot Q \cdot K^TMask⊙Q⋅KT 与 矩阵 VVV 想成，得到第一个 Self-Attention 的输出，这个输出只包含第一个单词的信息，且维度和输入矩阵维度相同。

Decoder

  Decoder 中的各个模块和 Encoder 模块是相同的结构，需要注意的一点是在第二个 Multi-Head Attention 中的 Q,K,VQ,K,VQ,K,V 取值上做了特殊处理。

  在这里，我们的 K,VK,VK,V 是来自于 Encoder 的输出 编码信息矩阵，由 编码信息矩阵 经过线性变换得到 K,VK,VK,V, 再根据上一个 Decoder Block 的输出经过线性变换得到 QQQ,如果是第一个 Decoder Block，则使用 Decoder 的输入矩阵进行计算得到 QQQ。这样做的好处是在 Decoder 的时候，每一个单词都可以利用到 Encoder 所有单词的信息 (这些信息无需 Mask)。
